# Business Requirements Document (BRD)

## 1. Project Overview

Blackjack Game is a browser-based card game application that allows users to play Blackjack against a virtual dealer.

---

## 2. Business Goal

The goal of the project is to create an interactive entertainment web application that demonstrates modern frontend development practices and provides users with a simple gaming experience.

---

## 3. Target Audience

The target audience includes:

* casual players
* students learning web technologies
* users interested in browser card games

---

## 4. Business Requirements

### BR-1 Accessibility

The application should be accessible through a web browser without additional software installation.

### BR-2 Easy Interaction

The user interface should be simple and intuitive.

### BR-3 Gameplay

The user should be able to:

* start a game
* draw cards
* stand
* receive the game result

### BR-4 Performance

The application should respond quickly to user actions.

### BR-5 GDPR Compliance

The system should notify users about cookies and consent storage.

---

## 5. Constraints

* The application does not support multiplayer mode.
* No real-money gambling functionality is included.
* The game works only with frontend technologies.

---

## 6. Expected Results

The project should provide:

* stable gameplay
* responsive user interface
* reusable React components
* project documentation

---

## 7. Success Criteria

The project is considered successful if:

* the game works correctly
* the user can complete gameplay without errors
* documentation is available
* Storybook components are implemented
